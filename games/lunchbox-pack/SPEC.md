# Lunchbox Pack — Build Spec (v1)

Read `game-design.md` first for the why. This file is the how. Pure static ES
modules, no framework, no build step. 2D DOM game (no three.js).

## Files

```
games/lunchbox-pack/
  index.html          splash + game shell
  css/style.css
  js/main.js          boot, splash, mode routing, audio unlock
  js/game.js          the round engine (all 3 modes)
  js/requests.js      request generation (pure functions, unit-testable)
  js/voice.js         recorded-clip player w/ speech fallback
  js/confetti.js      small DOM/canvas confetti burst (no three.js)
  data/foods.json     foods + characters (exists)
  assets/             bg.jpg, splash.jpg, lunchbox-open.png, lunchbox-closed.png
  assets/audio/       manifest.json + <key>.m4a clips (generated separately —
                      MAY NOT EXIST while you build; voice.js must fall back
                      cleanly to speech.js when manifest fetch fails)
  game.json           per-game manifest
  README.md           short; run from repo root
  ASSETS.md           provenance (AI-generated, see repo pattern in sound-sprouts)
```

## Shared imports (from js/ files: ../../../shared/…)

- `sfx.js`: pop, unpop, whoosh, sparkle, tada, silly, boing, tick, unlock()
- `speech.js`: speak(text), unlock(), stop()
- Fonts: Fredoka via css `../../../shared/fonts/fredoka-latin-600-normal.woff2`
- Food art: `../../shared/assets/foods/<id>.png` (document-relative from index.html)
- Character art: `../../shared/characters/<id>/portrait.png`
- HUD buttons: `../../shared/assets/ui/btn-home.png`, `btn-sound.png`

## voice.js contract

```js
await voice.init('./assets/audio/manifest.json');   // resolves even on 404
voice.say(key, fallbackText)  // recorded clip if present, else speech.speak(fallbackText)
voice.stop()
```
Manifest: `{ "<key>": { "file": "<key>.m4a", "dur": 1.23 }, ... }`. All keys and
their exact spoken text are in `data/lines.json` (same dir as manifest) — use its
text as the speech fallback so recorded and fallback voice always match.

## Layout (tablet landscape-first, matches concept screen)

- Fullscreen bg (assets/bg.jpg, cover).
- Top bar: home button (→ ../../ hub) left; sound-replay button right
  (replays current request); star progress row center (one star per box packed
  this session, 3 per session).
- Left panel: **food shelf** — scrollable grid of food cards (white rounded
  cards, food PNG + name label), 2 columns. Each session shows 10 shuffled
  foods INCLUDING every food the round's requests need (guarantee solvable).
- Center-left: **character** portrait with a speech bubble showing the current
  request as ART (food image, or group emoji chip, or color swatch/crunchy
  icon) — no reading required. Tapping character or bubble replays the voice.
- Right: **lunch box** (lunchbox-open.png, 532×620 — PORTRAIT: lid opens UP,
  the food tray is the BOTTOM HALF with a large green compartment left, pink
  top-right, yellow bottom-right). Size ~min(38vw, 58vh). Food dropped
  anywhere on the box lands in the next free slot. Slot positions (measured
  against the real art, % of image, center points), `SLOTS` const in game.js:
  `[{x:30,y:62,s:1.0},{x:66,y:54,s:.75},{x:60,y:75,s:.75},{x:22,y:56,s:.65},{x:38,y:68,s:.65}]`
  (x,y = center %, s = food scale) — keep as one obvious tunable const.
  The closed-lid image (lunchbox-closed.png, 579×509) overlays the same area
  on round end.
- Portrait orientation: shelf becomes a bottom row, box center, character top.

## Interaction

- **Drag**: pointer events (setPointerCapture) on food cards; a floating clone
  follows the pointer; drop over the box = attempt. Drop elsewhere = glide back.
- **Tap-tap fallback**: tap a food (it lifts + highlights), tap the box =
  same attempt; tap elsewhere cancels.
- Touch-action: none on cards during drag; manipulation elsewhere.
- ≥96px targets. No time pressure anywhere.

## Round engine

Session = 3 boxes (characters drawn without repeat, shuffled). Per box:
1. Character slides in; `voice.say('intro-<char>')`; empty open box.
2. Mode-specific requests (below). Correct food → flies into next slot,
   `sfx.pop` + `voice.say('yum-N')` (rotate 1-4). Wrong → `sfx.boing`, card
   wiggles + glides back, `voice.say('hint-wrong')`. NEVER punitive.
3. All requests done → `voice.say('lid')`, the closed-lid image drops over the
   box (bounce), tap it → confetti + `sfx.tada` + `voice.say('cheer')` (or
   'cheer-healthy'), star fills, next character. After 3rd box:
   `voice.say('bite')` + big star screen + replay/menu buttons.

### Mode: pack (Pack for Me!)
3 sequential requests per box, generated by requests.js:
- types: `food` (specific), `group`, `attr` (color or crunchy)
- Round 1 of session: [food, food, group] (easiest). Round 2: [food, group,
  attr]. Round 3: [group, attr, attr] (hardest).
- Weighted: 60% chance each request draws from the character's favorites.
- No duplicate correct-food answers within a box; attr/group requests accept
  ANY matching food not already packed.
- Request voice: `req-<food>` / `req-cat-<group>` / `req-attr-<attr>`.
- Bubble art: food image / group chip (emoji: fruit 🍎, veggie 🥦, main 🥪,
  drink 🥤, treat 🍪 + group color) / attr chip (color swatch circle, or
  "crunchy" = carrot icon w/ starburst).

### Mode: healthy (Healthy Helper)
One request: fill the 5-group meter — fruit, veggie, main, drink, treat.
Meter = 5 icon slots above the box that light up as packed. Any order. Voice
`mode-healthy` at start; on each correct: `grp-<group>` then a yum. Wrong =
food from an already-filled group → gentle wiggle-back + 'hint-wrong'.

### Mode: count (Count & Pack)
2 requests per box: `req-count-<n>` (n = 2..5, ramping) + bubble shows the
target food image with `×n`. The SAME food card can be dragged repeatedly
(don't remove it from the shelf); each drop lands a smaller copy in the box +
`voice.say('count-<i>')` counting up. Completing the count: yum + next.

## Feedback details

- Speech bubble chips also act as replay buttons.
- Idle nudge: 10s without progress → replay current request voice (once per
  request, don't nag).
- prefers-reduced-motion: no fly/bounce animations, instant placement.

## Debug hook (for automated QA — REQUIRED)

`window.LUNCH = { state(), pack(foodId) }` — state() returns
`{ screen, mode, character, requests: [...], currentRequest, packed: [...], stars }`;
pack(foodId) programmatically attempts packing that food (same code path as a
drop). Read-only otherwise. Mirrors sound-sprouts' `window.SPROUTS`.
